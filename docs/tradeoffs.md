# Tradeoff Log

### TO-001

- **Original Spec/Idea:** Move quickly toward broad PixPin feature parity across capture, pin, mark, OCR, and long capture.
- **Actual Implementation:** Prioritize stable baseline for static capture, image pin, and baseline mark tools before expanding to advanced modules.
- **Reasoning:** Baseline-first sequencing reduces regressions in daily workflows and keeps verification cost proportional for an open-source project with limited maintenance bandwidth.

### TO-002

- **Original Spec/Idea:** Match all PixPin documented features, including paid/VIP-only capabilities.
- **Actual Implementation:** Keep repository scope fully free/open-source; paid-only PixPin behaviors are not required for current parity target.
- **Reasoning:** Product direction is explicitly free and open-source. Scope is aligned to open implementations and avoids introducing paid-gated requirements as mandatory completion criteria.
